NRS is used to exchange presence and session information, which is needed to establish squads. Unlike other server infrastructure, there is currently no self-hostable NRS server due to abuse concerns. However, several NRS reimplementations are hosted publicly.

e-nrs

This instance supports 7.3.0 - 41.1.2 and proxying. You can configure SpaceNinjaServer to instruct your clients to connect to it like so:

  • Set "nrsAddresses" to ["e-nrs.source.wf"]
  • Set "dtls" to 99

o-nrs

This instance supports 35.5.0 - 40.0.5.1. You can configure SpaceNinjaServer to instruct your clients to connect to it like so:

  • Set "nrsAddresses" to ["o-nrs.source.wf"]
  • Set "dtls" to 99

c-nrs

This instance supports 41.0.x. You can configure SpaceNinjaServer to instruct your clients to connect to it like so:

  • Set "nrsAddresses" to ["c-nrs.source.wf:6500"]
  • Set "dtls" to 99
